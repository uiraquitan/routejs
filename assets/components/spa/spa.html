Propósito do Framework

Este mini-framework foi criado estrategicamente para desenvolvedores que conhecem o básico de HTML e CSS e desejam colocar seus projetos online de maneira organizada, veloz e estruturada. Ele elimina completamente a necessidade de duplicar linhas de código e tags repetidas (como cabeçalhos e menus).

Sua arquitetura limpa é ideal para realizar o deploy direto em ambientes tradicionais como cPanel ou serviços estáticos em nuvem, permitindo apresentar sistemas funcionais perfeitos antes mesmo de migrar o layout para um backend pesado ou duplicar arquivos de forma desnecessária.

Na linha 12, é apenas para que você crie somente um componente e ele possa ser chamado em qualquer lugar, sem precisar repetir o mesmo html copiando e salvando em várias pastas ou: componente1 na pasta 1, e componente1 na pasta2, sendo assim, podendo ser usado somente 1 e chamando ele em todas as outras estruturas de codigo. Acesse Componentes & Versões no menu a esquerda e veja um exemplo

Nas linhas 14 e 15, você tem a opção de escolher um arquivo requisitado como módulo ou diretamente, basta escolher uma das requisições de script.js

1: <!DOCTYPE html>
2: <html>
3: <head>
4:     <title>Meu Sistema SPA</title>
5:     <script>
6:         if ('serviceWorker' in navigator) {
7:             navigator.serviceWorker.register('/sw.js');
8:         }
9:     </script>
10: </head>
11: <body>
12:     <div include="./assets/components/sidebar.html"></div>
13:     <div id="app"></div>
14:     <script type="module" src="app.js"></script>
15:     <script type="module" src="./assets/js/script.js"></script>
16: </body>
17: </html>

sw.js (Urls Amigáveis)

self.addEventListener('fetch', (e) => {
    const url = new URL(e.request.url);
    if (e.request.mode === 'navigate' && !url.pathname.includes('.')) {
        e.respondWith(fetch('/index.html'));
    }
});
